# Use Vodafone Homebox FRITZ!Box 6670 as a pure cable modem in bridge mode
<img width="3464" height="1286" alt="CleanShot 2026-02-18 at 13 04 14" src="https://github.com/user-attachments/assets/492adecb-5f00-42be-8587-2b7091531bea" />

**DISCLAIMER:**
You can brick your box if you mess up the config file and don't generate a valid **checksum**!

## Intro

**Goal:** Enable bridge mode to use the FRITZ!Box only as a cable modem and use your own router / firewall which then gets the public IP via DHCP on the bridged port.

The Problem with most tutorials describing how to enable bridge mode is, they only show how to make the bridge mode visible in UI.

```
Lanbridges gui hidden = no; # default is yes
```

If you only bridge a port and connect your router with DHCP client, both the FRITZ!Box & your router **get the same public IP** on their WAN interface.

This won't work. Sometimes you might think "it's working" because you're able to ping hosts on the internet, but after a few minutes it stops working.\nMost of the time, it won't work at all.

Some reported, this might work if your ISP has "MaxCPE > 1" which basically means, they allow multiple MACs as CPE or sometimes also will provide more than one public IP.

**We need to ensure, that the FRITZ!Box won't dial-up / gets a public IP:**
<img width="3610" height="1460" alt="CleanShot 2026-02-18 at 13 04 35" src="https://github.com/user-attachments/assets/9f1ad316-3b5c-4344-8219-7104c83930c5" />

## Setup

* Vodafone Red Business Internet 300 Cable + static / fixed IP address and full dual stack (IPv4 + IPv6 /56 prefix)
  * **Internet only, no VoIP**
* Vodafone owned / rented FRITZ!Box
* FRITZ!OS: 8.03 (Vodafone)
* MikroTik router > ether1 with DHCP Client as WAN interface > LAN5 of FRITZ!Box (bridged port) > ether2 with DHCP Client to reach FRITZ!Box webinterface (192.168.178.1 > LAN1 of FRITZ!Box (LAN) >

## Export the config and edit the file
Some steps require physical access to press a button on the FRITZ!Box.
I recommend [setting up TOTP authentication](https://fritz.com/en/apps/knowledge-base/FRITZ-Box-7590/3282_Extra-confirmation-to-configure-certain-functions) for your user so you can enter a random 6-digit code instead generated by an authenticator app.

Export a config: System > Backup: This will export a file Einstellungen_FRITZ.Box_6670_Cable_284.08.03_11.02.26_1907**.export**
Make sure you keep an unaltered copy of that file and pen the file in a text editor.

## Generate new checksum and upload / restore the config
You can brick your box if you mess up the config file and don't generate a valid **checksum**!
We need to generate a new checksum for the altered config file.

### Restore the config
**Restore** the new config file (make sure it's the one with the **correct checksum**!
System > Backup > Restore & Reboot

## Enable Bridge Mode
Connect to the web interface via any LAN port (1-4) but the one you want to bridge (LAN5) and go to Internet > "Connection Type ?".\nYou should now see a new tab "Bridge Ports". Check LAN 5 and hit apply:
<img width="1870" height="608" alt="CleanShot 2026-02-18 at 13 07 56" src="https://github.com/user-attachments/assets/640fc1e0-fbfc-4082-a7ea-bef7d319d99c" />

## Connect your 3rd party router
* Connect LAN5 to the WAN port of your router with DHCP client running
* For IPv6 you need to request a prefix using DHCPv6 client with prefix hint `::/56`
<img width="2618" height="972" alt="CleanShot 2026-02-18 at 13 08 07" src="https://github.com/user-attachments/assets/89f7f10c-8f9a-442f-8561-40a701c2beb5" />
